# 翻訳ポリシー（日本語）

## 用語統一

| 英語 | 日本語訳 | 備考 |
|------|----------|------|
| skew | 軌道のずれ | レビュー指摘 PR #1 — "スキュー" は一般的でないため |
| pioneered by | 〜が考案した / 〜チームが考案した | "先駆けた" は意味が不明確なため |
| Absolute Encoder | 絶対エンコーダー | "アブソリュートエンコーダー"という表現は冗長で一貫性が無いため |

## 翻訳しない表現

- コードブロック内のすべての識別子・関数名・定数名・パラメーター名
- URL・リンク先テキスト
- PR 番号・issue 番号・GitHub ユーザー名（@mention）
- バージョン番号（例: `2025.8.0`）

## 文体

- 敬体（です・ます調）を使う
- 技術用語はカタカナ表記を優先し、日本語の慣用表現があればそちらを採用する

## その他の注意事項

- 画像パスは `ja/` 配下の相対パス（例: `../../.gitbook/assets/`）に調整する
- GitBook の `{% hint %}` / `{% embed %}` ブロックは構文を崩さない
