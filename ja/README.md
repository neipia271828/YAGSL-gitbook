# スワーブドライブドキュメントへようこそ

{% hint style="warning" %}
スワーブドライブが [CTRE](https://pro.docs.ctr-electronics.com/en/latest/index.html) の [Falcon500](https://store.ctr-electronics.com/falcon-500-powered-by-talon-fx/)/[Kraken](https://store.ctr-electronics.com/kraken-x60/)/[TalonFXS](https://store.ctr-electronics.com/products/talon-fxs)、[Pigeon2.0](https://store.ctr-electronics.com/pigeon-2/)、[CANCoder](https://store.ctr-electronics.com/cancoder/) のみで構成されている場合は、[Tuner X Swerve Drive Generator](https://pro.docs.ctr-electronics.com/en/latest/docs/tuner/tuner-swerve/index.html) を使用してください！

## 概要

YAGSL は、すべての FRC チームのために現役・元 BroncBotz メンターが開発したスワーブライブラリです。YAGSL の目標は、あらゆる経験レベルのチームが、あらゆる種類のスワーブモジュールを使って、誰もが知っている `DifferentialDrive` と同じくらい簡単にスワーブドライブを制御できるようにすることです。YAGSL はドキュメントが充実しており、積極的にメンテナンスされ、常に貢献を歓迎しています！

{% embed url="https://datawrapper.dwcdn.net/ZVxvE/13/" %}

## 私たちの理念

プログラムはスワーブドライブを中心に回るべきではありません。定数ファイルの中から正しいオプションを探すのに10分もかけるべきではありません。異なるロボットが同じコードで動作できるべきです。

## なぜ私たちは存在するのか？

世の中にあるスワーブドライブのコードのほとんどは、チームが自分のロボットに合わせて改変することを前提としたオリジナルのコードです。汎用的ではなく、動作させるまでに多大な時間と労力を要します。そのすべての労力を注いでも、バグが残ったり、加えた変更が期待通りに動かなかったりすることがあります。もっと良い方法があるはずです！

複数のロボットを持っていて、コードを変更せずに同じように動かしたいですか？YAGSL がそれを実現します！設定ディレクトリを作成するだけで、同じコードでロボットを動かせます！
