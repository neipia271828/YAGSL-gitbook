# はじめに

## スワーブドライブは複雑です！

スワーブドライブには多くのことがうまくいかない可能性があります。正しい情報がなければ、デバッグは長く困難なプロセスになりえます。

## スワーブドライブの設定には時間がかかります！

スワーブへの経験レベルによって、設定の経験は大きく異なります。スワーブモジュールを物理的に組み立てる時間も忘れずに考慮してください。

以前にスワーブのコードを使ったことがあれば、YAGSL の設定は比較的簡単だと思うでしょう。PID チューニングを含めて約 30 分で完了するかもしれません。スワーブドライブが動作したことが一度もない場合は、正しく設定するまでに**数週間**かかり、多くのサポートリクエストを送ることになるかもしれません。すべて動作しますが、忍耐は必須です！すべてをドキュメント化しようとしていますので、疑問があればここで確認してください！

## どれだけ望んでも、懇願しても、祈っても、生贄を捧げても、スワーブドライブが「勝手に動く」とは限りません

[CTRE Tuner X Swerve Generator](https://pro.docs.ctr-electronics.com/en/latest/docs/tuner/tuner-swerve/index.html) のようなツールは、デスクトップクライアントを使って COTS モジュールのアクションやプリセットデータに基づいてスワーブドライブの定数ファイルを設定してくれます。これは大きな助けになりますが、YAGSL が提供するものとは全く異なります（そして私が空き時間に作成できる量をはるかに超える作業量です）。スワーブドライブを正しくプログラムするには、WPILib と Java についてある程度の理解が必要です。知っておくべき例としては、[`XboxController`](https://github.wpilib.org/allwpilib/docs/release/java/edu/wpi/first/wpilibj/XboxController.html) から [`Joystick`](https://github.wpilib.org/allwpilib/docs/release/java/edu/wpi/first/wpilibj/Joystick.html) へのコントローラーの変更などがあります。
